# 编程式语言特性

编程式语言特性是[`vscode.languages.*`](/9.%20参考/1.%20VS%20Code%20API.md)API赋能的智能编辑特性。在Visual Studio Code中，有两种常用提供动态语言特性的方式。让我们以[悬浮提示](#显示悬浮提示)为例

```typescript
vscode.langauges.registerHoverProvider('javascript', {
    provideHover(document, position, token) {
        return {
            contents: ['Hover Content']
        };
    }
});
```

如您所见，[`vscode.languages.registerHoverProvider`](/9.%20参考/1.%20VS%20Code%20API.md)API提供了简易的方法，为JavaScript文件提供悬浮内容。在本扩展启用后，当您悬停在某些JavaScript代码后，VS Code查询所有JavaScript的[`HoverProvider`](/9.%20参考/1.%20VS%20Code%20API.md)，在悬浮组件中显示结果。[语言特性列表](#语言特性列表)和下面绘制的gif帮助您识别您的扩展需要的VS CodeAPI/LSP方法。

其他方式有实现适配[语言服务端协议](https://microsoft.github.io/language-server-protocol/)的语言服务端。工作方式为：

- 扩展提供JavaScript语言客户端和语言服务端。
- 语言客户端如其他VS Code扩展，在Node.js扩展宿主上下文中运行。当启用时，在另一个进程生成语言服务端进程并通过[语言服务端协议](https://microsoft.github.io/language-server-protocol/)交流。
- 您在VS Code中悬停在JavaScript代码上
- VS Code通知悬停的语言客户端
- 语言客户端查询语言服务端获取悬停结果，发送回VS Code
- VS Code在悬停组件中显示悬停结果

这个过程看起来更复杂，但它有两个主要的优势：
- 语言服务端可以由任意语言编写
- 语言服务端可以复用，为多个编辑器提供智能编辑特性

关于更多深入指南，前往[语言服务端扩展指南](7.%20语言服务端扩展指南.md)。

## 提供诊断

诊断是推断代码问题的方式。

![](https://code.visualstudio.com/assets/api/language-extensions/language-support/diagnostics.gif)

### 语言服务端协议
您的语言服务端向语言客户端发送`textDocument/publishDiagnostics`消息。消息包含资源URI的诊断项目数组。

**注意：**客户端不会向服务端请求诊断。服务端推送诊断信息给客户端。

### 直接实现

```typescript
let diagnosticCollection: vscode.DiagnosticCollection;

export function activate(ctx: vscode.ExtensionContext) : void {
    
    ···
    ctx.subscriptions.push(getDisposable());
    diagnosticCollection = vscode.languages.createDiagnosticCollection('go');
    ctx.subscriptions.push(diagnosticCollection);
    ···
}

function onChange() {
    let uri = document.uri;
    check(uri.fsPath, goConfig).then(errors => {
        diagnosticCollection.clear();
        let diagnosticMap: Map<string, vscode.Diagnostic[]> = new Map();
        errors.forEach(error => {
            let canonicalFile = vscode.Uri.file(error.file).toString();
            let range = new vscode.Range(error.line-1, error.startColumn, error.line-1, error.endColumn);
            let diagnostics = diagnosticMap.get(canonicalFile);
            if (!diagnostics) { diagnostics = []; }
            diagnostics.push(new vscode.Diagnostic(range, error.msg, error.severity));
            diagnosticMap.set(canonicalFile, diagnostics);
        });
        diagnosticMap.forEach((diags, file) => {
            diagnosticCollection.set(vscode.Uri.parse(file), diags);
        });
    })
}
```

#### 基础
回报诊断给打开的编辑器。此操作至少在每次保存时发生。诊断应当最好基于编辑器未保存内容计算。

#### 进阶
不仅给打开的编辑器，而且为打开的文件夹所有资源回报诊断，无论是否曾经在编辑器中打开。

## 显示代码补全方案

代码补全提供上下文感知建议给用户。

![](https://code.visualstudio.com/assets/api/language-extensions/language-support/code-completion.gif)

### 语言服务端协议
在对`initialize`方法的回复中，您的语言服务端需要声明提供补全，以及是否支持`completionItem\resolve`方法提供经过计算的补全项目的额外信息。

```json
{
    ···
    "capabilities": {
        "completionProvider": {
            "resolveProvider": "true",
            "triggerCharacters": [ '.' ]
        }
        ...
    }
}
```

### 直接实现

```typescript
class GoCompletionItemProvider implements vscode.CompletionItemProvider {
    public provideCompletionItems(
        document: vscode.TextDocument, position: vscode.Position, token: vscode.CancellationToken
    ):
    Thenable<vscode.CompletionItem[]> {
        ···
    }
}

export function activate(ctx: vscode.ExtensionContext): void {
    ···
    ctx.subscriptions.push(getDisposable());
    ctx.subscriptions.push(
        vscode.languages.registerCompletionItemProvider(
            GO_MODE, new GoCompletionItemProvider(), '.', '\"'));
    ···
}
```

#### 基础
您不需要处理提供器

#### 进阶
您支持处理提供器，计算用户选择的补全方案的额外信息。这条信息在选择的项目旁边显示。

## 显示内联补全
内联补全呈现为直接在编辑器中的多词元建议（*虚影文本*）。

![](https://code.visualstudio.com/assets/api/language-extensions/language-support/inline-completions.gif)

### 直接实现
```typescript
vscode.languages.registerInlineCompletionItemProvider({ language: 'javascript' }, {
    provideInlineCompletionItems(document, position, context, token) {
        const result: vscode.InlineCompletionList = {
            items: [],
            commands: [],
        };

        ···

        return result;
    }
});
```

您可以在[内联补全示例扩展](https://github.com/microsoft/vscode-extension-samples/tree/main/inline-completions)探索一个完整的示例。

#### 基础
仅为特定语言的一些常用模板提供基于本行内容的内联补全。

#### 进阶

基于整个文档内容或工作空间提供更复杂模板的内联补全

## 显示悬浮提示

悬浮提示显示关于鼠标指针下的符号/对象信息。通常为符号类型以及介绍。

![](https://code.visualstudio.com/assets/api/language-extensions/language-support/hovers.gif)

### 语言服务端协议
在对`initialize`方法的回复中，您的语言服务端需要声明支持悬浮提示。

```json
{
    ···
    "capabilities": {
        "hoverProvider": "true",
        ···
    }
}
```

另外，您的语言服务端需要回复`textDocument/hover`请求。

### 直接实现

```typescript
class GoHoverProvider implements HoverProvider {
    public provideHover(
        document: TextDocument, position: Position, token: CancellationToken):
        Thenable<Hover> {
    ···
    }
}

export function activate(ctx: vscode.ExtensionContext): void {
    ···
    ctx.subscriptions.push(
        vscode.languages.registerHoverProvider(
            GO_MODE, new GoHoverProvider()));
    ···
}
```


#### 基础
显示类型信息，如果可用显示文档。


#### 进阶
用代码相同的颜色样式为方法签名上色。
