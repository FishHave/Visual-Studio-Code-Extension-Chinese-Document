# 代码片段指南

[`contributes.snippets`](/9.%20参考/2.%20贡献点.md)贡献点帮助您打包代码片段进入Visual Studio Code扩展来分享。

[创建代码片段](https://code.visualstudio.com/docs/editing/userdefinedsnippets)主题涵盖了创建代码片段的所有信息。本指南/示例只是展示了您如何将您自己的代码片段变为可供分享的扩展。下面是建议的工作流：

- 使用`Snippets: Configure User Snippets`命令创建并测试您的代码片段。
- 在您的代码片段足够满意后，复制整个JSON文件到扩展文件夹，比如`snippets.json`
- 添加下面的贡献片段到`package.json`

```json
{
    "contributes": {
        "snippets": [
            {
                "language": "javascript",
                "path": "./snippets.json"
            }
        ]
    }
}
```

**提示：**使用下面的`package.json`配置，给您的扩展打上代码片段扩展的标签：

```json
{
    "categories": ["Snippets"]
}
```

您可以在https://github.com/microsoft/vscode-extension-samples/tree/main/snippet-sample找到完整的源代码。

## 使用TextMate片段
使用[yo code](/2.%20开始/1.%20您的第一个扩展.md)扩展生成器，您也可以添加TextMate片段（.tmSnippets）到VS Code安装中。生成器选项`New Code Snippets`让您指定包含多份.tmpSnippets文件的文件夹，所有的文件会被打包到VS Code代码片段扩展。生成器也支持Sublime代码片段（.sublime-snippets）。

最后生成器输出有两个文件：一份扩展清单`package.json`，包含整合代码片段进入VS Code所需元信息，以及包含转换为VS Code代码片段格式的`snippets.json`文件。

```bash
.
├── snippets                    // 整合进入VS Code
│   └── snippets.json           // 代码片段JSON文件
└── package.json                // 扩展清单
```

复制生成的代码片段文件夹到`.vscode/extensions`文件夹中的新文件夹，重启VS Code。
