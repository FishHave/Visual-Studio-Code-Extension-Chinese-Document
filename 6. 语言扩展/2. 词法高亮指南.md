# 词法高亮指南
词法高亮决定Visual Studio Code编辑器中源代码显示的颜色和样式。它负责给关键词上不同的颜色，比如说JavaScript中的`if`或者`for`关键字，它们的颜色和字符串和变量名不同。

词法高亮包括两个组成部分：

- [词元化](#词元化)：将文本分割为词元
- [主题化](#主题化)：使用主题或用户设置映射词元到特定颜色和样式

在深入细节前，可以通过尝试[词域检查器](#词域检查器)工具，探索源文件中呈现的词元，和它们各自匹配的主题规则。要同时查看语义和词法词元，在TypeScript文件中使用内建主题（例如，Dark+）。

## 词元化

文本词元化是指将文本分割为文段，并分类为不同的词元类型。

VS Code词元引擎基于[TextMate语法](https://macromates.com/manual/en/language_grammars)。TextMate语法由正则表达式的集合组成，并写入一个属性列表（XML）或JSON文件。VS Code扩展可以通过`grammars`贡献点贡献语法。

TextMate词元化引擎与渲染器运行在同一个进程，并且在用户打字时更新词元。词元用于词法高亮，同时也将源代码分为注释、字符串、正则表达式等区域。

从发布版1.43开始，VS Code也运行通过[语义词元提供器](/9.%20参考/1.%20VS%20Code%20API.md)贡献词元化。语义提供器通常由语言服务端实现，它们能够更深入理解源代码，可以在项目上下文中处理符号。例如，一个常量名可以使用常量高亮在整个项目中渲染，而不仅仅是在定义处。

基于语义词元的高亮是基于TextMate词法高亮的替代。语义高亮超越了词法高亮。因为语言服务端可能需要一小会加载和分析项目，语义高亮可能要在一小段延迟后出现。

本文专注于基于TextMate的词元化。语义词元化和主题化在[语义高亮指南](3.%20语义高亮指南.md)中解释。

### TextMate语法
VS Code使用[TextMate语法](https://macromates.com/manual/en/language_grammars)作为词法词元化引擎。最早用于TextMate编辑器，因为大量由开源社区创建和维护的语言包，它们已经被许多其他编辑器和IDE采用。

TextMate语法基于[Oniguruma正则表达式](https://macromates.com/manual/en/regular_expressions)，通常书写成属性列表或JSON。您可以在[这](https://www.apeth.com/nonblog/stories/textmatebundle.html)找到优秀的TextMate语法介绍，也可以看看已有的TextMate语法，了解更多它们的工作方式。

### TextMate词元和词域
词元是由一个或多个字符组成的同一程序元素。比如说`+`和`*`等运算符，`myVar`等变量名或者`"my string"`等字符串。

每个词元与定义词元上下文的词域相关联。词域是一个由点分割的标识符列表，指定当前词元的上下文。例如JavaScript中的`+`运算，拥有`keyword.operator.arithmetic.js`词域。

主题映射词域到颜色和样式以提供词法高亮。TextMate提供[通用词域列表](https://macromates.com/manual/en/language_grammars)，是很多主题的目标。为了让您的语法尽可能地受到广泛支持，尝试构建存在的词域，而不是定义一个新的。

词域可以嵌套，所以每个词元也与一系列父词域关联。下面的例子实用[词域检查器](#词域检查器)显示一个简单JavaScript函数中`+`运算符的词域层次。大部分词域在顶部列出，而更多通用父词语在下面列出。

![](https://code.visualstudio.com/assets/api/language-extensions/syntax-highlighting/scopes.png)

父词域的信息也用于主题化。当一个主题面向一个词域，所有具有此父词域的词元会被上色，除非主题也为它们个体的词域提供了一个更具体的上色。

### 配置括号匹配词域

一语言包括不参与括号匹配的词域，即使它们看起来很像括号。

通过两个属性配置括号匹配行为：

- `balancedBracketScopes`：定义参与括号匹配的词域。默认包括所有词域。
- `unbalancedBracketScopes`：定义排除括号匹配的词域。

```json
{
    "unbalancedBracketScopes": ["meta.scope.case-pattern.shell"]
}
```

### 贡献基本语法

VS Code支持json TextMate语法。通过`grammars`[贡献点](/9.%20参考/2.%20贡献点.md)贡献。

每个语法贡献点指定：语法应用的语言标识符、语法中词元最顶级词域的名称以及语法文件的相对路径。下面的示例展示了为虚构的`abc`语言的语法贡献。

```json
{
    "contributes": {
        "languages": [
            {
                "id": "abc",
                "extensions": [".abc"]
            }
        ],
        "grammars": [
            {
                "language": "abc",
                "scopeName": "source.abc",
                "path": "./syntaxes/abc.tmGrammar.json"
            }
        ]
    }
}
```

语法文件本身包括顶级规则。通常单独分为`patterns`字段，列出程序的顶级元素和定义每个元素的`respository`。语法中其他的规则可以实用`{"include": "#id"}`引用`repository`中的元素。

示例`abc`语法标记了字母`a`、`b`、`c`作为关键字，以及圆括号作为表达式的嵌套。

```json
{
    "scopeName": "source.abc",
    "patterns": [{"include": "#expression"}],
    "repository": {
        "expression": {
            "patterns": [{"include": "#letter"}, {"include": "#paren-expression"}]
        },
        "letter": {
            "match": "a|b|c",
            "name": "keyword.letter"
        },
        "paren-expression": {
            "begin": "\\(",
            "end": "\\)",
            "beginCaptures": {
                "0": {"name": "punctuation.paren.open"}
            },
            "endCaptures": {
                "0": { "name": "punctuation.paren.close" } 
            },
            "name": "expression.group",
            "patterns": [{ "include": "#expression" }]
        }
    }
}
```

语法引擎会尝试引用`expression`规则到文档中的所有文本。对于一个简单的程序：

```text
a
(
    b
)
x
(
    (
        c
        xyz
    )
)
(
    a
```

示例的语法缠上下面的词域（从左到右，从最底层到最高层词域列出）：

```text
a               keyword.letter, source.abc
(               punctuation.paren.open, expression.group, source.abc
    b           keyword.letter, expression.group, source.abc
)               punctuation.paren.close, expression.group, source.abc
x               source.abc
(               punctuation.paren.open, expression.group, source.abc
    (           punctuation.paren.open, expression.group, expression.group, source.abc
        c       keyword.letter, expression.group, expression.group, source.abc
        xyz     expression.group, expression.group, source.abc
    )           punctuation.paren.close, expression.group, expression.group, source.abc
)               punctuation.paren.close, expression.group, source.abc
(               punctuation.paren.open, expression.group, source.abc
a               keyword.letter, expression.group, source.abc

```

注意文本不是匹配规则其中之一，比如字符串`xyz`，被当前词域包括。文件末尾的圆括号是`expression.group`的一部分，尽管`end`规则不匹配，但是`end-of-document`在`end`规则之前适用。

### 嵌入语言

如果您的语法包含父语言中的嵌入语言，比如HTML中的CSS样式块，可以使用`embeddedLanguages`贡献点告诉VS Code把嵌入语言和父语言区分开。这保证了括号匹配、注释和其他基本语言特性在嵌入语言中也能按照预测工作。

`embeddedLanguages`贡献点映射嵌套语言中的词域到顶层语言的词域。在下面的示例，任何`meta.embedded.block.javascript`词域中的词元会被当做JavaScript内容：
```json
{
    "contributes": {
        "grammars": [
            {
                "path": "./syntaxes/abc.tmLanguage.json",
                "scopeName": "source.abc",
                "embeddedLanguages": {
                    "meta.embedded.block.javascript": "javascript"
                }
            }
        ]
    }
}
```

现在，如果您尝试在标记为`meta.embedded.block.javascript`中的词元注释代码或触发代码片段，它们会使用正确的`//`JavaScript格式注释和正确的JavaScript代码片段。

### 开发全新的语法扩展

如要快速创建新的语法扩展，使用[VS Code Yeoman模板](/2.%20开始/1.%20您的第一个扩展.md)，运行`yo code`，选择`New Language`选项：
![](https://code.visualstudio.com/assets/api/language-extensions/syntax-highlighting/yo-new-language.png)

Yeoman会带您走过一些基本问题，来搭建新扩展。对于创建新语法比较重要的问题是：

- `Language id`——您的语言的唯一标识符
- `Language name`——您的语言的人类可读名称
- `Scope names`——您的语法的TextMate根部词域名称

![](https://code.visualstudio.com/assets/api/language-extensions/syntax-highlighting/yo-new-language-questions.png)

生成器假定您想要同时定义语言和语法。如果您在为已经存在的语言创建语法，只需要将您的目标语言信息填入，确保删除了生成的`package.json`中的`languages`贡献点。

在回答了所有以上问题之后，Yeoman将以下面的结构创建新扩展：

![](https://code.visualstudio.com/assets/api/language-extensions/syntax-highlighting/generated-new-language-extension.png)

记住，如果您在贡献VS Code已经了解的语言语法，确认已经删除生成的`package.json`中的`languages`贡献点。
