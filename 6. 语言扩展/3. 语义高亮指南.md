# 语义高亮指南
如同[词法高亮指南](2.%20词法高亮指南.md)所介绍，语义高亮是对词法高亮的扩充。Visual Studio Code使用TextMate语法作为主要的词元化引擎。TextMate语法工作方式是将单个文件作为输入，基于正则表达式的词典规则将其分割。

语言服务端在项目上下文中，能够获知如何处理符号，语义词元化允许语言服务端基于此提供额外的词元信息。主题可以参与其中，使用语义词元改进语法上的词法高亮。编辑器将语义词元高亮覆盖于语法高亮上。

有了语义高亮：

## 语义词元提供器

如要实现语义高亮，语言扩展可以注册基于文档语言和/或文件名称的`语义词元提供器`。在需要语义词元时编辑器将对提供器发出请求。

```typescript
const tokenTypes = ['class', 'interface', 'enum', 'function', 'variable'];
const tokenModifiers = ['declaration', 'documentation'];
const legend = new vscode.SemanticTokensLegend(tokenTypes, tokenModifiers);

const provider: vscode.DocumentSemanticTokensProvider = {
    provideDocumentSemanticTokens(
        document: vscode.TextDocument
    ): vscode.ProviderResult<vscode.SemanticTokens> {
        // 分析文档，返回语义词元

        const tokensBuilder = new vscode.SemanticTokensBuilder(legend);
        //在第一行，第一到第五个字符为类声明
        tokensBuilder.push(
            new vscode.Range(new vscode.Position(1, 1), new vscode.Position(1, 5)),
            'class',
            ['declaration']
        );
        return tokensBuilder.build();
    }
};

const selector = { language: 'java', 'scheme': 'file' }; // 为所有来自本地文件系统的Java文档注册

vscode.languages.registerDocumentSemanticTokensProvider(selector, provider, legend);
```

承载语言服务端的功能的同时，语义词元提供器API分为两个派别：
- `DocumentSemanticTokensProvider`——一直采用整个文档作为输入。
    - `provideDocumentSemanticTokens`——提供文档的所有词元
    - `provideDocmentSemanticTokensEdits`——提供相较前一个回复的文档差别的所有词元。
- `DocumentRangeSemanticTokensProvider`——仅在一定范围内工作

提供器返回的所有词元都带有由词元类型、任意数量词元修饰符和词元语言的分类。

如同在上面的例子所看到的，它提供`SemanticTokensLegend`所要使用的提供器和修饰符。这让`provide`API将词元类型和修饰符作为索引返回给语例。

## 语义词元分类
语义词元提供器的输出包括词元。每个词元拥有一个范围和词元分类，这个词元分类描述了词元呈现为什么语法元素。如果词元是嵌入语言的一部分，分类也可以是语言名称。

语义词元类型和修饰符用来描述语法元素的种类。这个信息与[词法高亮指南](2.%20词法高亮指南.md)中介绍的TextMate词域类型，但是这是一个更专门和简洁的分类系统。

VS Code带有一系列语义词元类型和修饰符，供所有语义词元提供器使用。同时，语义词元提供器仍然可以自由定义新类型和修饰符，创建标准类型的子型。

### 标准词元类型和修饰符
标准类型和修饰符涵盖了许多语言共同的概念。然而每个语言也许会在某些类型和修饰符上使用不同的术语，通过依赖标准分类，主题作者就可以定义跨语言工作的主题规则。

### 自定义词元类型和修饰符
如果有必要，扩展可以通过扩展`package.json`中的`semanticTokenTypes`和`semanticTokenModifiers`贡献点声明新类型和修饰符或创建现有类型的子型：

```json
{
    "contributes": {
        "semanticTokenTypes": [
            {
                "id": "templateType",
                "superType": "type",
                "description": "A template type."
            }
        ],
        "semanticTokenModifiers": [
            {
                "id": "native",
                "description": "Annotates a symbol that is implemented natively"
            }
        ]
    }
}
```

在上面的示例中，扩展声明了一个新类型`templateType`和新修饰符`native`。通过将`type`写为超类型，`type`主题样式规则也会应用于`templateType`：

```json
{
    "name": "Red Theme",
    "semanticTokenColors": {
        "type": "#ff0011"
    }
}
```

上面所呈现的`semanticTokenColors`的值`#ff0011`会应用于`type`和其所有子类型，包括`templateType`。

除了自定义词元类型外，扩展可以定义它们与TextMate词域的映射关系。这些在[自定义映射](#自定义TextMate词域映射)小节中有介绍。注意自定义映射规则不会自动继承自其超类型。相反，子类型需要重新定义映射，最好是更细分的词域。

## 启用语义高亮

是否计算并高亮语义词元是由`editor.semanticHighlighting.enabled`设置决定的。它的允许值为`true`、`false`和`configuredByTheme`。

- `true`和`false`会对所有主题打开或关闭语义高亮。
- `configuredByTheme`是默认值，它让每个扩展自行控制是否启用语义高亮。VS Code上捆绑的所有主题（例如，默认“Dark+”主题）默认启用语义高亮。

依赖语义词元的语言扩展可以在`package.json`重载它们的语言默认值：

```json
{
    "configurationDefaults": {
        "[languageId]": {
            "editor.semanticHighlighting.enabled": true
        }
    }
}
```

## 主题化

主题化负责为词元指定颜色和样式。主题化规则由颜色主题文件（JSON格式）指定。用户也可以在用户设置中自定义主题化规则。

### 颜色主题中的语义上色

为了支持基于语义词元的高亮，颜色主题文件格式已经加入两种新属性。

`semanticHighlighting`属性定义主题是否准备好了语义词元的高亮。虽然默认为假，但是我们鼓励所有主题启用它。当设置`editor.semanticHighlighting.enabled`设置为`configuredByTheme`时使用此属性。

`semanticTokenColors`属性允许主题定义新的上色规则匹配语义词元提供器给出的语义词元类型和修饰符。

```jsonc
{
    "name": "Red Theme",
    "tokenColors": [
        {
            "scope": "comment",
            "settings": {
                "foreground": "#dd0000",
                "fontStyle": "italic"
            }
        }
    ],
    "semanticHighlighting": true,
    "semanticTokenColors": {
        "variable.readonly:java": "#ff0011"
    }
}
```

`variable.readonly:java`会调用选择器，具有形式`(*|tokenType)(.tokenModifier)*(:tokenLanguage)?`。

这些值表达了如果匹配规则应用的样式。它可以是表达前景色的字符串，也可以是在`tokenColors`中用于TextMate主题规则的对象，格式为`{ foreground: string, bold: boolean, italic: boolean, underline: boolean }`或者`{ foreground: string, fontStyle: string}`。

前景色需要遵循[颜色格式](/4.%20扩展指南/4.%20颜色主题.md#颜色格式)中介绍的颜色格式。不支持透明度。

下面是选择器和样式的其他例子：

- `"*.declaration"： { "bold": true } // 所有声明加粗`
- `"class:java": { "foreground": "#0f0", "italic": true } //Java类 `

如果没有规则匹配，或者主题没有`semanticTokenColors`字段（但是启用了`semanticHighlighting`）， VS Code会使用[语义词元词域映射](#语义词元词域映射)为给定语义词元求TextMate词域。词域会比较`tokenColors`中的TextMate主题化规则。

## 语义词元词域映射

为了让语义高亮在没有定义任何专门的语义规则主题中正常运作，并且作为自定义词元类型和修饰符的回滚，VS Code保留了一个从语义词元选择器到TextMate词域的映射。

如果主题让语义高亮启用，却没有包含给定语义词元的规则，就会使用这些TextMate词域找到TextMate主题化规则。

### 预定义TextMate词域映射
下面的表格列出了当前的自定义映射。

| **语义词元选择器** | **回退TextMate词域**| 
| - | - |
| `namespace` | `entity.name.namespace` |
| `type` | `entity.name.type` |
| `type.defaultLibrary` | `support.type` |
| `struct` | `storage.type.struct` |
| `class` | `entity.name.type.class` |
| `class.defaultLibrary` | `support.class` |
| `interface` | `entity.name.type.interface` |
| `enum` | `entity.name.type.enum` |
| `function` | `entity.name.function` |
| `function.defaultLibrary` | `support.function` |
| `method` | `entity.name.function.member` |
| `macro` | `entity.name.function.preprocessor` |
| `variable` | `variable.other.readwrite`、`entity.name.variable` |
| `variable.readonly` | `variable.other.constant` |
| `variable.readonly.defaultLibrary` | `support.constant` |
| `parameter` | `variable.parameter` |
| `property` | `variable.other.property` |
| `property.readonly` | `variable.other.constant.property` |
| `enumMember` | `variable.other.enummember` |
| `event` | `variable.other.event` |

### 自定义TextMate词域映射
扩展可以在`package.json`的`semanticTokenScopes`贡献点扩展此映射。

扩展有两种使用情形：

- 扩展定义了自定义词元类型和词元修饰符，提供TextMate词域作为主题未为语义词元类型或修饰符定义主题化规则时的回退。
```json
{
    "contributes": {
        "semanticTokenScopes": [
            {
                "scopes": {
                    "templateType": ["entity.name.type.template"]
                }
            }
        ]
    }
}
```

- TextMate语法提供器可以描述指定语言词域。着帮助包含特定语言主题化规则的主题。

```json
{
    "contributes": {
        "semanticTokenScopes": [
            {
                "language": "typescript",
                "scopes": {
                    "property.readonly": ["variable.other.constant.property.ts"]
                }
            }
        ]
    }
}
```
