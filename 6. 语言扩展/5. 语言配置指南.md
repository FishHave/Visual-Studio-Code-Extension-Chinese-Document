# 语言配置指南

[`contributes.languages`](/9.%20参考/2.%20贡献点.md)贡献点允许您定义控制下面声明性语言特性的语言配置：
- 注释切换
- 括号定义
- 自动闭合
- 自动包围
- 折叠
- 单词模式
- 缩进规则

这里有[语言配置实例](https://github.com/microsoft/vscode-extension-samples/tree/main/language-configuration-sample)，调控JavaScript的编辑体验。本指南解释了`language-configuration.json`的内容。

**注意：如果您的语言配置文件是，或以`language.configuration.json`结尾。您可以在VS Code中拥有自动补全和校验功能。**

```json
{
    "comment": {
        "lineComment": "//",
        "blockComment": ["/*", "*/"]
    },
    "brackets": [
        ["{", "}"],
        ["[", "]"],
        ["(", ")"]
    ],
    "autoClosingPairs": [
        { "open": "{", "close": "}"},
        { "open": "[", "close": "]"},
        { "open": "(", "close": ")"},
        { "open": "'", "close": "'", "notIn": ["string", "comment"] },
        { "open": "\"", "close": "\"", "notIn": ["string"] },
        { "open": "`", "close": "`", "notIn": ["string", "comment"] },
        { "open": "/**", "close": "*/", "notIn": ["string"] }
    ],
    "autoCloseBefore": ";:.,=}])>` \n\t",
    "surroundingPairs": [
        ["{", "}"],
        ["[", "]"],
        ["(", ")"],
        ["'", "'"],
        ["\"", "\""],
        ["`", "`"]
    ],
    "folding": {
        "markers": {
            "start": "^\\s*//\\s*#?region\\b",
            "end": "^\\s*//\\s*#?endregion\\b"
        }
    },
    "workPattern": "(-?\\d*\\.\\d\\w*)|([^\\`\\~\\!\\@\\#\\%\\^\\&\\*\\(\\)\\-\\=\\+\\[\\{\\]\\}\\\\\\|\\;\\:\\'\\\"\\,\\.\\<\\>\\/\\?\\s]+)",
    "indentationRules": {
        "increseIndentPattern": "^((?!\\/\\/).)*(\\{[^}\"'`]*|\\([^)\"'`]*|\\[^\\]\"'`]*)$",
        "decreseIndentPattern": "^((?!.*?\\/\\*).*\\*/)?\\s*[\\)\\}\\]].*$"
    }
}
```

## 注释切换
VS Code支持两种注释的注释切换。**切换行注释**和**切换块注释**。您可以指定`comment.blockComment`和`comments.lineComment`控制VS Code如何切换行/块注释。
```json
{
    "comment": {
        "lineComment": "//",
        "blockComment": ["/*", "*/"]
    }
}
```

`lineComment`属性为了向后兼容支持两种格式：
- 简单的行注释定义字符串值
- 允许配置注释行缩进行为的对象值。

```json
{
    "comment": {
        "lineComment": {
            "comment": "//",
            "noIndent": true
        },
        "blockComments": ["/*", "*/"]
    }
}
```

## 括号定义
当您移动指针到这里定义的括号，VS Code会将匹配的括号对一同高亮。

```json
{
    "brackets": [
        ["{", "}"],
        ["[", "]"],
        ["(", ")"]
    ]
}
```

并且，当您运行**转到括号**或者**选择括号所有内容**，VS Code会使用上面的定义找到最近的括号及其匹配对。

## 自动闭合
当您输入`'`，VS Code创建了一对单引号，并将您的指针放入中间：`'|'`。这个部分定义了这样的一对。

```json
{
  "autoClosingPairs": [
    { "open": "{", "close": "}" },
    { "open": "[", "close": "]" },
    { "open": "(", "close": ")" },
    { "open": "'", "close": "'", "notIn": ["string", "comment"] },
    { "open": "\"", "close": "\"", "notIn": ["string"] },
    { "open": "`", "close": "`", "notIn": ["string", "comment"] },
    { "open": "/**", "close": " */", "notIn": ["string"] }
  ]
}
```

`notIn`键在某些代码范围禁用这一特性。例如，当您在写下面的代码：

```javascript
// ES6's Template String
`ES6's Template String`;
```

单个引号不会自动闭合。

不需要`notIn`属性的对可以写成更简单的语法：
```json
{
    "autoClosingPairs": [
        ["{", "}"],
        ["[", "]"]
    ]
}
```

用户可以在`editor.autoClosingQuotes`和`editor.autoClosingBrackets`设置切换自动闭合行为。

### 在自动闭合前
VS Code默认只在指针左边为空格时自动闭合对。所以当您在下面的JSX代码输入`{`，您不会触发自动闭合：

```javascript
const Component = () =>
    <div className={>
                   ^ 默认不触发自动闭合
    </div>
```

然而，这个定义可以覆写这种行为：
```json
{
    "autoCloseBefore": ";:.,=}])>` \n\t"
}
```
现在当您在`>`后输入`{`，VS Code会自动闭合`}`。

## 自动包围
当您选中VS Code的一片区域，并输入开括号，VS Code会用一对括号包围选中的内容。这项特性叫做自动包围，这里您可以为指定语言定义自动包围对：
```json
{
  "surroundingPairs": [
    ["{", "}"],
    ["[", "]"],
    ["(", ")"],
    ["'", "'"],
    ["\"", "\""],
    ["`", "`"]
  ]
}

```

用户可以在`editor.autoSurround`设置切换自动包围行为。
## 折叠
在VS Code，折叠要么基于缩进定义，要么基于贡献折叠范围提供器定义：
- 缩进定义通过标记折叠：如果给定语言没有可用的范围提供器，或用户已经设置`editor.foldingStrategy`为`indentation`，就使用基于缩进的折叠。折叠区域开始于拥有较小缩进的行，它之后有一个或多个紧随行，结束于相同或更小缩进的行。忽略空行。另外，语言配置可以定义开始和结束标记。即`folding.markers`中的`start`和`end`正则表达式。当找到匹配的行，将创建标记对中的折叠范围。折叠标记不为空，并且通常类似于`//#region`和`//#endregion`。

下面的JSON为`//#region`和`//#endregion`创建了折叠标记。

```json
{
  "folding": {
    "markers": {
      "start": "^\\s*//\\s*#?region\\b",
      "end": "^\\s*//\\s*#?endregion\\b"
    }
  }
}
```

- 语言服务端折叠：语言服务端用一系列折叠范围回复[`textDocument/foldingRange`](https://microsoft.github.io/language-server-protocol/specification#textDocument_foldingRange)请求，VS Code会把这个范围作为折叠标记渲染。子[编程式语言特性](6.%20编程语言特性.md)主题了解更多有关语言服务端协议折叠支持。

## 单词模式
`workPattern`定义编程语言中将什么视作一个单词。如果设置了`workPattern`，代码提示功能将使用此设置决定单词边界。注意这个设置不会影响单词相关编辑器指令，这些指令由`editor.wordSeparators`编辑器设置控制。

```json
{
  "wordPattern": "(-?\\d*\\.\\d\\w*)|([^\\`\\~\\!\\@\\#\\%\\^\\&\\*\\(\\)\\-\\=\\+\\[\\{\\]\\}\\\\\\|\\;\\:\\'\\\"\\,\\.\\<\\>\\/\\?\\s]+)"
}
```

## 缩进规则
`indentationRules`定义了编辑器如何调整您输入、粘贴或移动的当前行或下一行的缩进。

```json
{
  "indentationRules": {
    "increaseIndentPattern": "^((?!\\/\\/).)*(\\{[^}\"'`]*|\\([^)\"'`]*|\\[[^\\]\"'`]*)$",
    "decreaseIndentPattern": "^((?!.*?\\/\\*).*\\*/)?\\s*[\\)\\}\\]].*$"
  }
}
```

例如，`if (true) {`匹配`increaseIndentPattern`，如果您在开括号`{`随后输入<kbd>Enter</kbd>，编辑器会自动进行一次缩进，您的代码最后会是：
```javascript
if (true) {
    console.log();
```

除了`increaseIndentPattern`和`decreaseIndentPattern`，还有两条其他的缩进规则：

- `indentNextLinePattern` - 如果一行匹配了此模式，**只有下一行**会进行一次缩进。
- `unIndentedLinePattern` - 如果一行匹配了此模式，它的缩进将不会改变，也不会用其他规则演算。

如果编程语言没有设置缩进规则，编辑器会在用开括号结束的行自动缩进，并在输入闭括号后自动取消缩进。这里的括号由`brackets`定义。

注意，`editor.formatOnPaste`设置由[`DocumentRangeFormattingEditProvider`](/9.%20参考/1.%20VS%20Code%20API.md)控制，不会受自动缩进影响。

## 回车规则
`onEnterRules`定义了一系列规则，会在编辑器按下<kbd>Enter</kbd>时演算。
```json
{
    "onEnterRules": [
        {
            "beforeText": "^\\s*(?:def|class|for|if|elif|else|while|try|with|finally|except|async).*?:\\s*$",
            "action": { "indent": "indent" }
        }
    ]
}
```

当按下<kbd>Enter</kbd>，这段文本的之前、之后或指针上方的行会检查下面的属性：

- `beforeText`（必需）。匹配指针之前文本的正则表达式（限制于本行）。
- `afterText`。匹配指针之后文本的正则表达式（限制于本行）。
- `previousLineText`。匹配指针上方行的正则表达式。

如果所有指定的属性匹配，会匹配此规则，其他`onEnterRules`不会被演算。`onEnterRule`可以指定下面的操作：

- `indent`（必需）。`none, indent, outdent, indentOutdent`之一。
    - `none`意思新行会继承当前行的缩进。
    - `indent`意思是新行会相对本行缩进。
    - `outdent`意思是新行会相对本行取消缩进。
    - `indentOutdent`意思是会插入两条新行，一条缩进，第二条取消缩进。
- `appendText`。在新行和缩进后插入一段字符串。
- `removeText`。从新行缩进移除的字符数。
