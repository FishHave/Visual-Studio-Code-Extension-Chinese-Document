# Web扩展
Visual Studio Code编辑器可以在浏览器中运行。比如说在GitHub浏览仓库，或者拉取请求，通过按下`.`（点键）进入`github.dev`用户界面。在Web使用VS Code，安装的扩展会在浏览器中的扩展宿主运行，叫做“Web扩展宿主”。可以在Web扩展宿主中运行的扩展叫做“Web扩展”。

Web扩展与常规扩展使用相同的结构，但考虑到不同的运行时，不会使用为Node.js运行时而编写的扩展的代码。Web扩展仍然可以访问完整的VS Code API，但不再能使用Node.js API和模块加载，而是被限制在浏览器沙盒中，因此，对比普通扩展有所[限制](#Web扩展主文件)。

Web扩展运行时也受VS Code桌面端支持。如果您决定创建一个Web扩展，它支持[Web VS Code](https://code.visualstudio.com/docs/setup/vscode-web)（包括`vscode.dev`和`github.dev`），也支持桌面端，和[GitHub Codespaces](https://code.visualstudio.com/docs/remote/codespaces)等服务。

## Web扩展详解
Web扩展的[结构类似常规扩展](/2.%20开始/2.%20扩展解剖.md)。扩展清单（`package.json`）定义扩展源代码的入口文件，以及声明扩展贡献。

对于Web扩展来说，[主入口文件](#Web扩展主文件)由`browser`属性定义，而不是与常规扩展一样由`main`属性定义。

`contributes`属性与常规扩展工作方式相同。

下面的例子展示了一个简单的Hello World扩展的`package.json`，这个扩展仅运行在Web扩展宿主（仅拥有`browser`入口）：
```json
{
    "name": "helloworld-web-sample",
    "displayName": "helloworld-web-sample",
    "description": "HelloWorld example for VS Code in the browser",
    "version": "0.0.1",
    "publisher": "vscode-samples",
    "repository": "https://github.com/microsoft/vscode-extension-samples/helloworld-web-sample",
    "engines": {
        "vscode": "^1.74.0"
    },
    "categories": ["Other"],
    "activationEvents": [],
    "browser": "./dist/web/extension.js",
    "contributes": {
        "commands": [
            {
                "command": "helloworld-web-sample.helloworld",
                "title": "Hello World"
            }
        ]
    },
    "scripts": {
        "vscode:prepublish": "npm run package-web",
        "compile-web": "webpack",
        "watch-web": "webpack --watch",
        "package-web": "webpack --mode production --devtool hidden-source-map"
    },
    "devDependencies": {
        "@types/vscode": "^1.59.0",
        "ts-loader": "^9.2.2",
        "webpack": "^5.38.1",
        "webpack-cli": "^4.7.0",
        "@types/webpack-env": "^1.16.0",
        "process": "^0.11.10"
    }
}
```

> **注意**：如果您的扩展面向高于1.74的VS Code，您必须在`activationEvents`中单独列出`onCommand:helloworld-web-sample.helloworld`

仅有`main`入口，而没有`browser`的扩展不是Web扩展。它们会被Web扩展宿主忽略，并且在扩展视图中不可下载。

![](https://code.visualstudio.com/assets/api/extension-guides/web-extensions/extensions-view-item-disabled.png)

仅有声明性贡献（仅`contributes`，没有`main`或者`browser`）可以是Web扩展。它们可以在[Web VS Code](https://code.visualstudio.com/docs/setup/vscode-web)中安装运行，并且不需要扩展作者作出任何修改。扩展声明性贡献的例子有主题、语法和代码片段。

扩展可以同时具有`browser`和`main`入口使其可以在浏览器和Node.js运行时运行。[更新已有的扩展为Web扩展](#更新已有的扩展为Web扩展)小节展示了如何将扩展迁移为能在两种运行时工作。

[Web扩展的启用](#Web扩展的启用)小节列举了用来决定扩展额能否在Web扩展宿主加载的条件。

### Web扩展主文件
Web扩展主文件由`browser`属性定义。Web扩展的脚本寄宿在[浏览器WebWorker](https://developer.mozilla.org/zh-CN/docs/Web/API/Web_Workers_API)环境。它被限制在浏览器Worker沙盒，相比运行在Node.js运行时的普通扩展有所限制。

- 不支持导入或需求其他模块。`importScripts`同样不可用。所以，代码必须打包为单个文件。
- [VS Code API](/9.%20参考/1.%20VS%20Code%20API.md)可以通过`require('vscode')`的方式加载。这仍然可以使用，因为我们给`require`开了后门，但是这个后门不可以用于加载额外的扩展文件或者Node模块。它只能使用`require('vscode')`。
- Node.js全局成员和库，比如`process`、`os`、`setImmediate`、`path`、`util`、`url`在运行时中不可用。然而，它们可以使用类似Webpack这样的工具添加。[Webpack配置](#Webpack配置)小节解释了这是如何做到的。
- 开放的工作空间或者文件夹在虚拟文件系统上。对工作空间文件的访问权限需要经过VS Code在`vscode-workspace-fs`[文件系统](/9.%20参考/1.%20VS%20Code%20API.md)API。
- [扩展上下文](https://code.visualstudio.com/api/references/vscode-api)位置（`ExtensionContext.extensionUri`）和存储位置（`ExtensionContext.storageUri`,`globalStorageUri`）也可以运作在虚拟文件系统，并且需要经过`vscode.workspace.fs`。
- 要访问Web资源，必须使用[Fetch](https://developer.mozilla.org/zh-CN/docs/Web/API/Fetch_API)。访问的资源需要支持[跨源资源共享](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Guides/CORS)（CORS）
- 创建子进程或者运行可执行文件是不可能的。然而，可以通过[Worker](https://developer.mozilla.org/en-US/docs/Web/API/Worker)API创建Web Worker。这用于如[Web扩展语言服务端协议](#Web扩展语言服务端协议)小节介绍的运行语言服务端。
- 作为常规扩展，`activate/deactivate`函数需要通过`exports.activate = ...`的方式导入。

## 开发Web扩展
谢天谢地，TypeScript和Webpack这样的工具可以避免许多浏览器运行时限制，允许您用常规扩展相同的方式编写Web扩展。Web扩展和常规可以可以生成自同一份源代码。


例如，由`yo code`[生成器](https://www.npmjs.com/package/generator-code)创建的`Hello Web Extension`仅在构建脚本方面有所不同。通过使用**调试：选择并开始调试**命令提供的提供的启动配置，您可以像传统Node.js扩展一样，运行和调试生成的扩展。

## 创建Web扩展
为了搭建新的Web扩展，使用`yo code`，选取**New Web Extension**。确保`generator-code`是最新版本（>=generator-code@1.6）。如需更新生成器和yo，运行`npm i -g yo generator-code`。

创建扩展由扩展源代码（显示hello world通知的命令）、`package.json`清单文件和webpack或者esbuild配置文件组成。

让事情简单一些，我们假定您使用`webpack`作为打包工具。在文章的最后，我们会解释选择`esbuild`会有什么不同。

- `src/web/extension.ts`是扩展的入口源代码文件。这和常规的hello扩展是显著的不同之处。
- `package.json`是扩展清单
    - 它使用`browser`属性指向入口文件
    - 它提供脚本：`compile-web`、`watch-web`和`package-web`，用来编译、观测和打包。
- `webpack.config.js`是webpack配置文件，编译并打包扩展源代码为单个文件。
- `.vscode/launch.json`包含运行Web扩展的启动配置和VS Code桌面端Web扩展宿主的测试（不再需要设置`extensions.webWorker`）。
- `.vscode/task.json`包括启动配置使用的构建任务。它使用`npm run watch-web`，并依赖于webpack特别的`ts-webpack-watch`问题配对器。
- `.vscode/extensions.json`包括提供问题配对其的扩展。这些扩展需要安装使启动配置能够工作。
- `tsconfig.json`定义匹配`webworker`运行时的编译选项。

[helloworld-web-sample](https://github.com/microsoft/vscode-extension-samples/tree/main/helloworld-web-sample)源代码与生成器创建的那个相同。

### Webpack配置
Webpack配置文件由`yo code`自动生成。它从您的扩展中打包源代码为单个JavaScript文件，并在Web扩展宿主中加载。

之后我们会解释如何使用esbuild作为打包工具，但是我们从Webpack开始

[webpack.config.js](https://github.com/microsoft/vscode-extension-samples/blob/main/helloworld-web-sample/webpack.config.js)

```javascript
const path = require('path');
const webpack = require('webpack');

/** @typedef {import('webpack').Configuration} WebpackConfig **/
/** @type WebpackConfig */
const webExtensionConfig = {
    mode: 'none', // 这让源代码尽量与原版接近（打包时我们将其设置为'production'）
    target: 'webworker', // 扩展运行在Webworker上下文
    entry: {
        extension: './src/web/extension.ts', // Web扩展主文件的源
        'test/suite/index': './src/web/test/suite/index.ts' // Web扩展测试运行器的源
    },
    output: {
        filename: '[name].js',
        path: path.join(__dirname, './dist/web'),
        libraryTarget: 'commonjs',
        devtoolModuleFilenameTemplate: '../../[resource-path]'
    },
    resolve: {
        mainFields: ['browser', 'module', 'main'], // 在导入的Node模块中查找`browser`入口
        extensions: ['.ts', '.js'], // 支持ts文件和js文件
        alias: {
            // 提供Node模块和源文件的其他实现
        },
        fallback: {
            // Webpack 5 不再自动替代Node.js的核心模块
            // 参见 https://webpack.js.org/configuration/resolve/#resolvefallback
            // 以获取Node.js核心模块的替代列表
            assert: require.resolve('assert')
        }
    },
    module: {
        rules: [
            {
                test: /\.ts$/,
                exclude: /node_modules/,
                use: [
                    {
                        loader: 'ts-loader'
                    }
                ]
            }
        ]
    },
    plugins: [
        new webpack.ProvidePlugin({
            process: 'process/browser' // 提供全局`process`变量的后门
        })
    ],
    externals: {
        vscode: 'commonjs vscode' // 忽略，因为它不存在
    },
    performance: {
        hints: false
    },
    devtool: 'nosources-source-map' // 创建源映射，指向原版源代码
};
module.exports = [webExtensionConfig];
```
